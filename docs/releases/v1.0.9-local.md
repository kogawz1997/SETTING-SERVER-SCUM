# SETTING SERVER SCUM v1.0.9-local

รอบนี้เป็น local release สำหรับปิดงาน hardening ก่อนปล่อย portable zip ให้คนอื่นโหลดไปใช้จริง จุดหลักคือให้ตัว zip ตรวจตัวเองหลังแตกไฟล์, launcher บอกปัญหาแบบอ่านรู้เรื่อง, catalog มีชื่อ/หมวด/rarity ที่แม่นขึ้น และหน้า Graph ลากเชื่อม ref ได้เข้าใจกว่าเดิม

## ของใหม่

- เพิ่ม release zip smoke test: สร้าง zip, แตกไฟล์ด้วย `Expand-Archive`, ตรวจ `portable-manifest.json`, ตรวจไฟล์จำเป็น, ตรวจ version และกันไฟล์ส่วนตัวหลุดเข้า package
- เพิ่ม gate ใน `release:quality` และ GitHub Actions ให้ต้องผ่าน `npm run package:portable:zip-smoke` ก่อนปล่อย release
- Portable launcher แสดงขั้นตอนเป็นภาษาคน เช่น Checking portable files, Checking Node.js, Opening browser และมี `How to fix:` เมื่อเจอปัญหา
- Item Catalog เพิ่ม curated metadata สำหรับปืน/กระสุน/ยา/อาหาร/รถ เช่น AK-47, M82A1, SVD Dragunov, Block 21, adrenaline, canteen, car battery และ ammo box สำคัญ
- Graph editor เพิ่มเส้น preview ตอนลากจาก spawner ไป node/ref พร้อม label ปล่อยเพื่อเชื่อม ref และ staged summary หลังปล่อย ก่อน Preview/Apply จริง
- เพิ่ม test coverage สำหรับ catalog metadata และ graph drag connect flow ใน UI smoke

## วิธีใช้

โหลด zip จาก Assets, แตกไฟล์ลงโฟลเดอร์ใหม่ แล้วเปิด `Start SETTING SERVER SCUM.exe`

ถ้า Windows บล็อกไฟล์ `.exe` ให้ใช้ `Start SETTING SERVER SCUM.cmd` แทน

ถ้า launcher แจ้งไฟล์ขาด ให้แตก zip ใหม่ทั้งชุด ห้าม copy เฉพาะ exe ไปเปิดเดี่ยวๆ เพราะ launcher ต้องใช้ไฟล์ server, public, scripts และ manifest ในชุดเดียวกัน

## หมายเหตุ

- ยังเป็น local-first tool ไม่ใช่ SaaS
- ก่อนแก้ไฟล์จริง ให้ตั้งค่า path, backup folder, Nodes folder และ Spawners folder ให้ครบ แล้วตรวจ Customer Ready/Diagnostics ก่อน
- Graph editor รอบนี้เน้น Spawner -> Ref ที่ปลอดภัยก่อน ยังไม่ใช่ full graph editor แบบลากทุก relationship ทุกชนิด
