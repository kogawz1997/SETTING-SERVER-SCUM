# SETTING SERVER SCUM v1.0.6-local

Release date: 2026-04-21

This release closes the local readiness polish after v1.0.5. It focuses on safer first-run checks, better diagnostics, and a more reliable portable test/release flow.

## For Normal Use

1. Download `SETTING-SERVER-SCUM-v1.0.6-local.zip` from GitHub Release.
2. Extract the zip.
3. Double-click `Start SETTING SERVER SCUM.exe`.
4. If Windows blocks the EXE, double-click `Start SETTING SERVER SCUM.cmd`.
5. Open Dashboard or Settings and review Startup Doctor before editing real server files.

## Highlights

- Adds Startup Doctor panels to Dashboard and Settings so config path, Nodes folder, Spawners folder, backup, permissions, and reload/restart commands are easier to understand before real use.
- Adds structured operation logs in `logs/operations.jsonl` and includes recent operation logs in support bundles.
- Makes Playwright smoke/regression tests launch their own isolated local server on a free port, avoiding stale `localhost:3000` conflicts.
- Fixes Loot Studio kit template saving by restoring backend item name/probability normalization.
- Updates Loot Studio regression coverage for autofix and quick-fix flows using real malformed loot fixtures.
