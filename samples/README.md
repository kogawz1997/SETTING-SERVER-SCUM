# Sample Workspace

This folder is a safe sandbox for demos, onboarding, release checks, and validation tests.

- Use `samples/scum-workspace/WindowsServer` as the SCUM config folder when you want to try the app without touching real server files.
- The `broken-cases` folder contains intentionally bad loot samples used by validation and autofix demos.
- Run `npm run sample:smoke` to confirm the sample workspace still parses and still contains useful broken cases.
